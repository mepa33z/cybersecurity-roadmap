This course item focuses on safe browsing practices and application ecosystem security, highlighting how to protect apps, devices, and business data from cyber threats.

App and Software Security Concerns:

Mobile apps can be vulnerable due to weak passwords, malware, or poor design, risking access to personal and business data.
Rooting or jailbreaking phones increases functionality but also security risks; app stores restrict insecure apps to maintain safety.

Device and Software Protection Measures:

Strong passwords, multi-factor authentication (MFA), antivirus/malware software, VPNs, and regular updates help secure devices and software.
Physical security of PCs, secure browsing on HTTPS sites, and proper software configuration reduce hacking risks.

Business Data Security Practices:

Businesses face threats like ransomware and zero-day attacks targeting sensitive data and intellectual property.
Limiting access, educating employees on security, and using secure file-sharing solutions help protect corporate data and prevent insider threats.


This course item covers the risks and security concerns associated with using public browsing and public Wi-Fi networks.

Public Browsing Risks:

Public Wi-Fi networks are often unencrypted, allowing eavesdroppers to intercept data sent and received on the network.
Hackers can install malware, hijack sessions, or steal credentials when devices connect to unsecured public networks.

Types of Data and Threats:

Data tracked on public Wi-Fi can be sold to advertisers or malicious parties.
Common threats include session hijacking, shoulder surfing, and interception of sensitive information like banking credentials.

Safe Browsing Practices:

Use mobile hotspots or cellular connections instead of public Wi-Fi when possible.
Employ firewalls, strong passwords, antivirus software, VPNs, and keep software updated to protect devices.
Verify websites use HTTPS (indicated by URL prefix and lock icon) for encrypted connections, but remain cautious as HTTPS does not guarantee site safety.
Be cautious with social networking and instant messaging on public networks, avoiding sharing sensitive data and suspicious links.


This course item focuses on safe browsing practices, including configuring browsers securely, identifying secure websites, and managing browser add-ons.

Browser Security Configuration:

Modern browsers like Chrome, Edge, Firefox, and Safari are common but often not securely configured by default.
Older browsers used security zones to control website permissions, but modern browsers do not use zones; enabling features like ActiveX can introduce risks.

Browser Add-ons and Cookies:

Toolbars, once common, are mostly obsolete due to performance issues and security risks.
Plug-ins are site-specific and largely unsupported now; extensions add functionality but can be targets for attackers.
Cookies track user activity; types include session, persistent, authentication, tracking, first-party, third-party, and zombie cookies, with varying privacy implications.

Website Security and Browser Updates:

SSL certificates authenticate websites and enable encrypted connections (HTTPS), helping ensure site legitimacy.
Browsers update automatically to patch vulnerabilities, which is critical since browsers are common attack targets.
Proper management of cookies, extensions, and updates helps maintain data security during browsing.
